## 二、顶部状态栏设计

```
┌─────────────────────────────────────────────────────────────────┐
│  🎯 Phase 2 · 文案撰写系统          [📊 创意数据库：47条] [v1.2▼]│
│                                       ⚠️ 冷启动模式（建议继续积累） │
└─────────────────────────────────────────────────────────────────┘

── 数据库状态徽章 ──────────────────────────────────────────────────
  ┌────────────────────┐
  │ 📊 47 条 / 200    │  ← 进度条填充
  │ ██████░░░░░░░░░░  │
  └────────────────────┘

状态判断逻辑：
  < 50 条  →  ⚠️ 冷启动模式（橙色徽章）
  50-199条 →  📈 成长模式（蓝色徽章）
  ≥ 200 条 →  ✅ 正常模式（绿色徽章）
```

**数据库状态提示文案**：
- 冷启动：`⚠️ 创意数据库 47 条（冷启动模式，建议继续用插件积累至 200 条）`
- 成长模式：`📈 创意数据库 127 条（成长模式，已启用数据库检索）`
- 正常模式：`✅ 创意数据库 312 条（正常模式，数据驱动文案生成）`

## 十一、状态机

### 11.1 任务状态流

```
IDLE（首页）
  │ 点击"开始生成"
  ▼
GENERATING（生成中）
  │ 完成
  ▼
REVIEW（评审页）◄──────────┐
  │                         │ 修改后重新提交
  │ 选择 ≥1 条
  ▼                         │
BRIEF_PREVIEW（Brief 预览）  │
  │                         │
  │ 提交 Brief              │
  ▼                         │
SUBMITTED（已提交）──────────┘
  │ 返回修改
  ▼
COMPLETED（完成）
```

### 11.2 文案卡片状态

```
UNREVIEWED（默认）→  ⭐ 采用  ✏️ 修改  ❌ 淘汰
       │
       ├─ 采用 → ADOPTED（绿色高亮）
       ├─ 修改 → MODIFIED（橙色标记，显示修改后内容）
       └─ 淘汰 → REJECTED（灰色，折叠到待定Tab）
```

## 十二、边界情况与降级

| 情况 | UI 处理 |
|------|--------|
| 数据库为空（0条） | 顶部橙色警告条：`⚠️ 创意数据库为空，将使用行业通用公式模板。建议先用 Phase 1 插件积累数据。` |
| Claude 返回非法 JSON | Toast 提示：`生成失败，Claude 返回格式异常`。自动重试一次，仍失败显示错误 + 重试按钮 |
| 网络断线 | 全屏遮罩 + `网络连接已断开，请检查网络后重试`。断线重连后自动恢复上一次状态 |
| 生成超时（>30s） | 显示：`生成时间过长，可能遇到网络问题。[终止] 或 [继续等待]` |
| Brief 提交失败 | Toast 错误提示，保留 Brief 预览页，可重新提交 |
| Prompt 版本切换中 | 全局禁用操作，顶部显示 `⏳ Prompt 版本切换中...` |
