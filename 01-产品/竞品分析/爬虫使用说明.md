# 竞品数据采集脚本 — 使用说明

脚本位于 `竞品分析/scripts/`，采集结果自动写入 `竞品分析/数据/<平台>/` 作为 Obsidian 笔记，可在 [[竞品数据库]] 表格视图中统一查看。

---

## 一、安装依赖

```bash
cd 竞品分析/scripts

# 安装 Python 依赖（建议 Python 3.11+）
pip install -r requirements.txt

# 安装 Playwright 浏览器内核
playwright install chromium
```

---

## 二、配置 `config.py`

打开 `config.py`，填入以下信息：

### 关键词和平台

```python
KEYWORDS = ["无核山楂", "山楂片"]   # 修改为你要搜索的关键词
PLATFORMS = ["amazon", "1688", "temu", "shopee"]  # 去掉不需要的平台
```

### Amazon PA-API（可选，仅采集 Amazon 需要）

> [!info] 申请地址
> https://affiliate-program.amazon.com/
> 注册联盟账号 → 申请 PA-API Access Key

```python
AMAZON_API = {
    "access_key": "AKXXXXXXXXXX",
    "secret_key": "xxxx",
    "partner_tag": "yourstore-20",
    "region": "US",
}
```

也可以通过环境变量传入（更安全）：
```bash
set AMAZON_ACCESS_KEY=AKXXXXXXXXXX
set AMAZON_SECRET_KEY=xxxx
set AMAZON_PARTNER_TAG=yourstore-20
```

### 代理配置（可选）

| 平台 | 是否需要代理 | 说明 |
|------|------------|------|
| Amazon | ❌ 不需要 | 直接调用官方 API |
| 1688 | ⚠️ 若在海外则需要中国 IP | 国内网络可不配置 |
| Temu | ⚠️ 可选 | 初期可不配置，被封再加 |
| Shopee | ⚠️ 建议配置新加坡 IP | 或直接用海外网络 |

```python
PROXY = {
    "1688":   "http://user:pass@cn-proxy:port",
    "temu":   "",   # 暂不配置
    "shopee": "",   # 暂不配置
}
```

### 飞书多维表格配置（可选，使用 `--output feishu` 时必填）

> [!info] 飞书应用创建步骤
> 1. 打开 https://open.feishu.cn/app → 点击「创建企业自建应用」
> 2. 进入应用 → 「权限管理」→ 搜索并添加：`bitable:app` 和 `bitable:record:write`
> 3. 「版本管理与发布」→ 申请上线（或直接在测试模式使用）
> 4. 在飞书云文档新建多维表格，URL 格式：`/base/<app_token>?table=<table_id>`

```python
FEISHU = {
    "app_id":     "cli_xxxxxxxxxx",       # 开放平台应用 App ID
    "app_secret": "xxxxxxxxxxxxxxxx",     # 开放平台应用 App Secret
    "app_token":  "Bkxxxxxxxxxxxxxx",     # 多维表格 URL 中 /base/ 后面的字符串
    "table_id":   "tblxxxxxxxxxx",        # 多维表格 URL 中 ?table= 后面的字符串
}
```

或通过环境变量传入：
```bash
set FEISHU_APP_ID=cli_xxxxxxxxxx
set FEISHU_APP_SECRET=xxxxxxxxxxxxxxxx
set FEISHU_APP_TOKEN=Bkxxxxxxxxxxxxxx
set FEISHU_TABLE_ID=tblxxxxxxxxxx
```

> [!warning] 还需要将应用添加为多维表格的协作者
> 在飞书多维表格右上角 → 分享 → 邀请「应用」→ 输入应用名称 → 给予编辑权限

## 三、首次登录 1688（仅限 1688 平台）

1688 需要登录状态才能获取完整数据。**首次运行前**，执行：

```bash
python main.py --login-1688
```

这会打开一个 Chromium 浏览器窗口，**手动完成登录**（包括扫码/手机验证码）。登录成功后，在命令行按回车，Cookie 自动保存到 `cookies/1688_cookies.json`。

> [!warning] Cookie 有效期
> 1688 Cookie 通常有效 30 天左右。过期后重新执行上述命令即可。
> Cookie 文件已加入 `.gitignore`，不会同步到 GitHub。

## 四、运行采集

```bash
# 采集 config.py 中配置的所有关键词 × 所有平台（写入 Obsidian，默认）
python main.py

# 仅采集特定关键词
python main.py --keyword 无核山楂

# 仅采集特定平台
python main.py --platform amazon
python main.py --platform 1688 temu

# 单关键词 + 单平台（调试用）
python main.py --keyword 无核山楂 --platform amazon

# 所有平台
python main.py --platform all

# ── 飞书输出模式 ──────────────────────────────────────────────
# 写入飞书多维表格（需先完成 config.py 飞书配置）
python main.py --output feishu

# 同时写入 Obsidian 和飞书
python main.py --output both

# 指定平台 + 飞书输出
python main.py --keyword 无核山楂 --platform amazon --output feishu
```

## 七、常见问题

### Q: Amazon 报错 `ValueError: Amazon PA-API 凭证未配置`
填入 `config.py` 中的 `AMAZON_API` 凭证，或设置环境变量。

### Q: 1688 显示 Cookie 失效
重新执行 `python main.py --login-1688`，手动登录后保存新 Cookie。

### Q: Temu/Shopee 采集到空列表
可能触发了反爬机制。尝试：
1. 将 `BROWSER_HEADLESS = False` 改为显示浏览器，观察页面行为
2. 配置住宅代理
3. 增大 `DELAY_MIN` / `DELAY_MAX` 延迟

### Q: Playwright 报错 `Executable doesn't exist`
执行：`playwright install chromium`

### Q: 飞书写入报错 `code=...`
常见原因：
1. `app_token` 或 `table_id` 填写有误（注意区分大小写）
2. 应用未授权 `bitable:record:write` 权限
3. **应用未被添加为多维表格的协作者**（最常见！）→ 在飞书表格右上角「分享」→ 邀请应用并给予编辑权限

### Q: 飞书报错 `缺少飞书 SDK`
执行：`pip install lark-oapi`

### Q: 飞书字段名不匹配
飞书多维表格的字段名必须与 `output/to_feishu.py` 中 `_item_to_fields()` 的 key 完全一致。建议在飞书建表时直接复制字段说明中的中文字段名。

## 八、技术架构

```
main.py  (--output obsidian|feishu|both)
  ├── scrapers/amazon.py      ← PA-API 5.0（无需浏览器）
  ├── scrapers/taobao_1688.py ← Playwright + Cookie
  ├── scrapers/temu.py        ← Playwright + Stealth
  ├── scrapers/shopee.py      ← Playwright + Stealth
  └── output/
      ├── to_obsidian.py     ← 转换为 .md 笔记（--output obsidian）
      └── to_feishu.py       ← 调用飞书 Open API（--output feishu）
```

> [!note] Amazon PA-API 注意事项
> 当前使用 PA-API 5.0。Anthropic 公告显示该 API 将于 **2026-04-30 废弃**，届时需迁移到 Amazon Creators API。届时更新 `scrapers/amazon.py` 即可，其余模块不受影响。
