# 芯威霆企业网站 - 后台管理系统方案

## 背景

现有前端网站（React + Vite + Tailwind）由 Figma 导出，所有内容（招聘信息、公司数据、联系表单）全部硬编码在组件中。HR 无法自行更新内容，联系表单也没有后端处理。需要搭建一个轻量后台，让 HR 能直接管理最常变动的内容。

---

## 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 后端 | Express.js | 轻量、生态成熟 |
| 数据库 | better-sqlite3 | 零配置，单文件，Windows 免安装 |
| 认证 | JWT (jsonwebtoken + bcryptjs) | 无状态，无需 session store |
| 管理前端 | React 18 + Vite + Tailwind CSS | 与主站一致，复用 UI 组件 |
| 跨域 | cors | 开发环境前后端分离 |

---

## 功能模块（按 HR 优先级排序）

### MVP（首期实现）

1. **招聘管理** — 职位的增删改查、上下架开关、排序
2. **留言管理** — 查看联系表单提交，已读/未读标记，删除
3. **首页仪表盘** — 在招职位数、未读留言数、最近留言

### Phase 2（后续扩展）

4. **员工福利管理** — 编辑福利卡片内容
5. **网站数据设置** — 团队规模、满意度等 key-value 指标

## 数据库设计

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  display_name TEXT NOT NULL DEFAULT '',
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE jobs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  department TEXT NOT NULL,
  location TEXT NOT NULL DEFAULT '福州仓山区',
  type TEXT NOT NULL DEFAULT '全职',
  experience TEXT NOT NULL DEFAULT '不限',
  description TEXT NOT NULL,
  is_active INTEGER NOT NULL DEFAULT 1,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  phone TEXT DEFAULT '',
  email TEXT DEFAULT '',
  content TEXT NOT NULL,
  is_read INTEGER NOT NULL DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE benefits (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  sort_order INTEGER NOT NULL DEFAULT 0,
  is_active INTEGER NOT NULL DEFAULT 1,
  updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  label TEXT NOT NULL DEFAULT '',
  updated_at TEXT DEFAULT (datetime('now'))
);
```

首次启动自动将前端现有硬编码数据写入为种子数据。默认管理员：`admin / xinweiting2026`。

## API 设计

### 认证

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/auth/login` | 登录，返回 token |
| PUT | `/api/auth/password` | 修改密码 |

### 招聘管理

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/jobs` | 公开 | 获取在招职位（前端网站调用）|
| GET | `/api/jobs?all=true` | 需登录 | 获取全部职位含已下架 |
| POST | `/api/jobs` | 需登录 | 新增职位 |
| PUT | `/api/jobs/:id` | 需登录 | 编辑职位 |
| PUT | `/api/jobs/:id/toggle` | 需登录 | 上架/下架切换 |
| DELETE | `/api/jobs/:id` | 需登录 | 删除 |

### 留言管理

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| POST | `/api/messages` | 公开 | 前端表单提交 |
| GET | `/api/messages` | 需登录 | 留言列表（含未读数） |
| GET | `/api/messages/:id` | 需登录 | 查看详情（标记已读） |
| DELETE | `/api/messages/:id` | 需登录 | 删除 |

### 网站设置 (Phase 2)

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/settings` | 公开 | 获取全部设置 |
| PUT | `/api/settings` | 需登录 | 批量更新 |

## 前端网站改造

对现有前端页面做最小改动：用 `useEffect` + `fetch` 从 API 加载数据，API 不可用时回退到硬编码默认值。

需改动的文件：

- `app/pages/Careers.tsx` — 职位列表和福利从 API 获取
- `app/pages/Contact.tsx` — 表单提交 POST 到 API

---

## 管理界面页面

- **登录页** — 居中卡片，用户名 + 密码
- **仪表盘** — 欢迎语 + 统计卡片（在招职位数、未读留言、总留言）+ 最近 5 条留言
- **招聘管理** — 表格展示职位，支持新增/编辑/删除/上下架
- **留言管理** — 表格展示留言，未读高亮，点击查看详情
- **网站设置** — key-value 表单（Phase 2）

全部中文界面。

## 启动方式

```bash
cd 芯威霆企业网站-admin
npm install
npm run dev          # 同时启动 Express(3001) + Vite(5174)
```

生产环境：`npm run build` 后 Express 同时托管静态文件，单进程运行。
