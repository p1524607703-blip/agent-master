# 选品工具可行性分析

> [!info] 项目概述
> 跨平台选品工具，Vue 前端网页版。目标：从多个社交/广告平台发现有潜力的跨境电商选品机会。
> 分析由 Claude + Codex（MiniMax-M2.5）联合完成，日期：2026-03-16

## 数据来源评估

| 平台 | 官方 API | 接入难度 | 数据价值 | 综合评分 |
|------|---------|---------|---------|---------|
| Reddit | ✅ 免费官方 API | 低 | 真实用户讨论，选品信号强 | **9/10** |
| Google Trends | ✅ pytrends 库 | 低 | 趋势权威，搜索量数据 | **9/10** |
| Facebook 广告库 | ✅ Ad Library API（免费） | 中 | 爆款广告直接参考 | **7/10** |
| TikTok | ⚠️ Research API（审核严） | 高 | 爆款商品信号极强 | **4/10** |
| 小红书 | ❌ 无官方 API | 极高 | 国内种草，违规风险大 | **1/10** |

## 各平台技术详解

### Google Trends

- `pytrends` 非官方库可直接调用，免登录
- 后端 Python 封装接口供 Vue 调用
- **风险**：高频请求触发限流，需做缓存层

### Facebook 广告库

> [!tip] 最高价值数据源
> Ad Library API 完全免费，可查全球在投广告。竞品广告 = 直接选品参考。

- 只需注册 Meta 开发者账号，无需企业资质
- 可查关键词、地区、广告时长等维度

### Reddit

- 官方 API 60 请求/分钟（免费），足够日常使用
- 重点版块：r/entrepreneur、r/ecommerce、r/dropship
- **最容易接入，建议第一个开发**

### TikTok

- Research API 需企业资质 + 审核，周期 2-4 周
- **替代方案**：Apify TikTok 抓取器（按次付费）
- 或抓取公开热搜榜作为轻量替代

### 小红书

> [!warning] 暂缓接入
> 无官方 API，爬虫违反用户协议，法律风险较高。建议先跳过，后续考虑人工采集或数据合作。

## 开发优先级

```
第一阶段（MVP）  →  Reddit + Google Trends
第二阶段        →  Facebook 广告库
第三阶段        →  TikTok（第三方服务）
第四阶段        →  小红书（暂缓）
```

## 综合评分

| 维度 | 评分 | 备注 |
|------|------|------|
| 技术可行性 | 7/10 | 前三平台完全可做 |
| Vue 前端难度 | 中等 | 数据展示、筛选、趋势图 |
| 后端难度 | 中等 | Python/Node 聚合层 |
| 合规风险 | 低 | 仅用官方 API 时 |

> [!success] 结论
> **项目可行**，建议从 Reddit + Google Trends 起步，2 周内可出 MVP。Facebook 广告库作为第二阶段核心亮点。
