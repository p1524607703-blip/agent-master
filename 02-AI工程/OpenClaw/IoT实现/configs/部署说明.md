# go2rtc + Home Assistant 首次部署步骤

> [!important] 版本锁定
> go2rtc 使用 **v1.9.13**，不要用 latest 或 v1.9.14+。
> v1.9.14 存在 CS2 协议 i/o timeout 回归（[issue #2048](https://github.com/AlexxIT/go2rtc/issues/2048)）。

---

## 前置条件

- WSL2 + Docker Engine 已安装（参考 [[环境检查报告]]）
- 小米 C700 摄像头已连入局域网，能正常在米家 App 中查看
- 已准备好以下信息：

| 信息 | 说明 | 获取方式 |
|------|------|---------|
| 米家账号 | 手机号或邮箱 | 米家 App → 我的 → 账号 |
| 米家密码 | 登录密码 | — |
| 摄像头 IP | 局域网 IP 地址 | 米家 App → 设备 → 设置 → 网络信息 |
| 设备 DID | 设备唯一标识 | 米家 App → 设备 → 设置 → 关于，或通过 go2rtc WebUI 获取 |

---

## 第一步：填写配置

编辑 `go2rtc.yaml`，替换四个占位符：

```yaml
streams:
  c700_living_room:
    - xiaomi://手机号:密码@192.168.1.xxx?did=设备DID&model=chuangmi.camera.81ac1
```

> [!tip] 密码中的特殊字符
> 如果密码包含 `@`、`:`、`?` 等特殊字符，需要 URL 编码（如 `@` → `%40`）。

---

## 第二步：启动服务

```bash
cd /path/to/configs  # 进入 docker-compose.yml 所在目录

# 拉取镜像并启动
docker compose up -d

# 查看运行状态
docker compose ps

# 查看 go2rtc 日志（确认摄像头连接）
docker compose logs -f go2rtc
```

---

## 第三步：处理米账号验证码

> [!warning] 首次登录可能触发验证码
> 小米账号首次在新设备/IP 登录时，可能触发滑块验证码或短信验证码。
> go2rtc 无法自动处理验证码。

**解决方法：**

1. 先在同一台机器的浏览器中访问 [https://account.xiaomi.com](https://account.xiaomi.com)
2. 用相同的米家账号登录，完成验证码验证
3. 登录成功后，go2rtc 再次连接时通常不会再触发验证码
4. 如果仍然触发，尝试在 go2rtc WebUI（`http://localhost:1984`）中使用「Add Camera → Xiaomi」方式登录

---

## 第四步：验证 go2rtc 连接

1. 浏览器打开 `http://localhost:1984`
2. 在 Streams 列表中应能看到 `c700_living_room`
3. 点击进入，确认视频流正常播放
4. RTSP 地址：`rtsp://localhost:1984/c700_living_room`

---

## 第五步：配置 Home Assistant

1. 浏览器打开 `http://localhost:8123`
2. 首次访问需要创建管理员账号
3. 完成初始化引导后，进入 **设置 → 设备与服务 → 添加集成**
4. 搜索 `go2rtc` 或 `Generic Camera`，添加摄像头：
   - Still Image URL: `http://localhost:1984/api/frame.jpeg?src=c700_living_room`
   - Stream Source: `rtsp://localhost:1984/c700_living_room`
5. 在 HA 用户资料中生成「长期访问令牌」，供 OpenClaw Skill 调用

---

## 第六步：测试 HA REST API

```bash
# 获取摄像头截图（替换 YOUR_HA_TOKEN）
curl -s -o snapshot.jpg \
  -H "Authorization: Bearer YOUR_HA_TOKEN" \
  "http://localhost:8123/api/camera_proxy/camera.c700_living_room"

# 验证截图文件
file snapshot.jpg
# 应输出：JPEG image data
```

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| go2rtc 日志 `i/o timeout` | 使用了 v1.9.14+ | 降级到 v1.9.13 |
| 设备列表为空 | 米账号验证未通过 | 先在浏览器登录米家账号 |
| 视频卡顿 | 网络带宽不足或 WiFi 信号弱 | 确保摄像头 WiFi 信号良好 |
| HA 无法连接 go2rtc | Docker 网络隔离 | 使用 `network_mode: host` 或确认端口映射 |
| `EOF` 错误 | 型号不兼容 | 检查 [[go2rtc已知问题清单]] |
