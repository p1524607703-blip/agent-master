# 02 多轮对话连贯性测试

> [!info] 理论依据
> **MT-Bench（Zheng et al., 2023）** 的核心创新：两轮对话，第二轮基于第一轮的回答提出 follow-up，考察模型是否真正"理解"了上下文，而不只是独立回答。

## 核心测试能力

- **上下文记忆**：记住之前说过的内容
- **指代消解**：理解"它"、"这个"、"刚才"指什么
- **立场一致性**：不在不同轮次前后矛盾
- **渐进修正**：用户纠错后能正确更新
- **长对话退化**：10轮后是否开始漂移

---

## 标准测试对话组

### 【对话组 1：概念深化】

```
轮1：解释一下什么是"锚定效应"
轮2：好的，那它在电商定价中是怎么被利用的？给个具体例子
轮3：如果我是消费者，怎么识别并避免被这个效应影响？
```
考察：知识连贯 + 应用迁移 + 实用建议

---

### 【对话组 2：角色扮演一致性】

```
轮1：你现在扮演一位 1920 年代的上海侦探，说话风格要有年代感
轮2：（用现代语气问）你觉得人工智能会让你失业吗？
轮3：好了，不用角色扮演了，正常回答我：什么是情感计算
```
考察：角色保持 → 处理"出戏"问题 → 干净切换

---

### 【对话组 3：渐进纠错】

```
轮1：万里长城是什么时候开始建的？
轮2：我看到一个资料说是公元前 7 世纪就开始了，你的说法不太准确
轮3：好，那秦始皇统一之前各诸侯国修的那些算不算长城的一部分？
```
考察：接受纠错 + 知识更新 + 不过分自我辩解

---

### 【对话组 4：偏好记忆】

```
轮1：我是一个素食主义者，给我推荐一些高蛋白食物
轮2：这些都不错，帮我设计一个一周的早餐计划
轮3：我对坚果过敏，重新调整一下计划
轮4：好的，把这个计划变成购物清单格式
```
考察：跨轮次约束叠加（素食 + 无坚果 + 格式）

---

### 【对话组 5：矛盾处理】

```
轮1：帮我写一封强硬的投诉信，要求全额退款
轮2：算了，其实我觉得只要部分退款也行，改得温和一点
轮3：不对，我越想越气，还是要强硬，但要更有理有据
```
考察：接受反转 + 不质疑用户 + 风格还原

---

## 评分维度（每组 5 分）

| 维度 | 权重 |
|------|------|
| 上下文引用准确 | 30% |
| 逻辑连贯无矛盾 | 30% |
| 风格/立场一致 | 20% |
| 自然过渡不生硬 | 20% |
