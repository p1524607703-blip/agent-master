# 01 指令遵循测试

> [!info] 理论依据
> **IFEval（Zhou et al., 2023）** 将指令拆解为可验证的原子约束（如字数、格式、关键词），通过程序自动判断是否遵从。AlpacaEval 则用模型胜率衡量整体指令跟随质量。

## 测试维度分解

| 子类型 | 描述 | 难度 |
|--------|------|------|
| 格式约束 | 指定字数、段落数、列表格式 | 低 |
| 输出结构 | 要求 JSON / Markdown / 表格 | 中 |
| 角色扮演 | 持续保持特定身份 | 中 |
| 多步骤指令 | 同时满足 3+ 个条件 | 高 |
| 否定指令 | "不要用某词"、"不要举例子" | 高 |
| 元指令遵从 | 用户明确说"按我的格式来" | 高 |

---

## 标准测试题库

### 【格式约束】

**T1-01**
```
用恰好 3 个要点总结一下人工智能的主要风险，每个要点不超过 20 字，用"-"开头。
```
评分标准：✓ 恰好3条  ✓ 每条≤20字  ✓ 用"-"开头

**T1-02**
```
给我写一个 100 字左右（90-110字）的产品介绍，产品是一款智能台灯。
```

---

### 【输出结构】

**T1-03**
```
把以下信息整理成 JSON 格式：
姓名：张三，年龄：28，职业：设计师，城市：上海
```
评分标准：✓ 合法 JSON  ✓ 字段名合理  ✓ 类型正确（数字不加引号）

**T1-04**
```
用 Markdown 表格列出 iPhone 16、华为 Mate 60、三星 S24 的对比，包含：屏幕尺寸、电池容量、起售价三列。
```

---

### 【否定指令】

**T1-05**
```
解释"量子纠缠"，不要用任何比喻，不要提"魔法"或"神奇"，不超过 150 字。
```
评分标准：✓ 无比喻  ✓ 无禁词  ✓ ≤150字

**T1-06**
```
写一首关于秋天的诗，但不要出现"落叶"、"枫叶"、"金黄"这三个词。
```

---

### 【多步骤 & 复合】

**T1-07**
```
请做以下三件事：
1. 给"人工智能"写一个 50 字定义
2. 列出 3 个应用领域
3. 最后用一句话说明你认为最大的风险
每部分用标题分隔。
```

**T1-08（高难度）**
```
以一位持怀疑态度的记者身份，写一篇 200 字左右的评论，主题是"AI 能否替代医生"。
要求：
- 开头必须是一个反问句
- 至少引用一个虚构的数据（需标注"数据来源：虚构"）
- 结尾必须是开放性问题
```

---

## 评分 Rubric

| 分数 | 标准 |
|------|------|
| 5 | 所有显式约束均满足，输出质量高 |
| 4 | 满足 80% 约束，有小瑕疵 |
| 3 | 满足核心约束，遗漏次要约束 |
| 2 | 仅满足 50% 约束 |
| 1 | 基本忽略约束，自由发挥 |
| 0 | 完全不相关或拒绝 |
