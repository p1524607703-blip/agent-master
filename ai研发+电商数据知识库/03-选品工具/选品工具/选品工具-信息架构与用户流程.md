## 二、新手用户核心流程

```
访问首页
    │
    ▼
弹出新手引导（首次访问）："3步找到好产品"
    │
    ▼
Step 1：选目标市场 + 类目（北美/欧洲 × 3C/家居/服饰）
    │
    ▼
Step 2：AI 问答收集约束（预算 / 经验 / 目标利润）
    │
    ▼
Step 3：生成选品报告（AI 一句话结论 + 可视化图表）
    │
    ├──→ 加入收藏夹
    ├──→ 开启定时监控（推送到微信/飞书/邮件）
    └──→ 切换专业模式查看完整数据
```

### 流程关键节点

| 阶段 | 用户动作 | 系统响应 | 关键体验 |
|------|---------|---------|---------|
| 入门 | 首次访问 | 弹出新手引导，可跳过 | 不强制，有退出口 |
| 输入 | 选市场/类目 | 实时更新推荐列表 | 即时反馈 |
| 分析 | 回答 3~5 个问题 | 收集选品约束条件 | 进度条显示 |
| 输出 | 确认分析 | 骨架屏加载 + AI 生成报告 | 加载有进度感 |
| 决策 | 阅读结论 | 可视化图表 + 通俗文字 | 结论前置 |
| 后续 | 收藏/开启监控 | 持久化 + 推送配置 | 一键完成 |

## 三、风险清单

### 🔴 高风险

| 风险 | 问题 | 应对措施 |
|------|------|---------|
| 数据合规 | Facebook/TikTok 爬取可能违反平台 ToS | 优先官方 API；标注数据来源；加免责声明 |
| AI 幻觉 | AI 推荐的品类已饱和或数据失真 | 多源交叉验证；标注数据更新时间；显示置信度 |
| 数据准确性 | 多平台数据延迟不一致 | 每个数据块独立标注更新时间 |

### 🟡 中风险

| 风险 | 问题 | 应对措施 |
|------|------|---------|
| OpenClaw 稳定性 | Agent 操作网页，前端改动导致 agent 失效 | **提供稳定的 agent 专用 API 接口**，不依赖页面 DOM |
| 推送失败 | 微信/飞书/邮件推送失败率高 | 重试机制 + 失败告警 + 备用渠道 |
| B 端权限 | 多成员权限混乱 | 角色分级（管理员/分析师/查看者）+ 操作日志 |

### 🟢 改进点

| 风险 | 问题 | 应对措施 |
|------|------|---------|
| Reddit 语言障碍 | 内容全英文，新手看不懂 | AI 翻译摘要层（中文一句话总结） |
| 冷启动 | 新用户首屏空看板体验差 | 默认推荐内容兜底，不依赖历史数据 |
| 加载性能 | 多数据源聚合慢 | 骨架屏 + 后端预计算 + 增量加载 |

## 四、关键设计决策说明

> [!note] Claude vs Codex 分歧与采纳理由
> - **看板三段漏斗**（采纳 Codex）：收藏→跟踪→决策池，符合用户真实决策路径
> - **AI 模式渐进展开**（采纳 Claude）：专业数据按需展开，避免硬切换导致的割裂感
> - **Agent 专用 API**（双方共同发现）：OpenClaw 不应依赖页面 DOM，必须有稳定的数据接口层
> - **Reddit 翻译摘要**（双方盲区，Claude 补充）：国内新手用户看不懂英文内容，需要 AI 摘要层
