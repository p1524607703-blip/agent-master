# TikTok 鞋类广告 AI 全链路工作流

> [!info] 背景
> 团队20人，各部门分工协作。当前最卡点：**创意同质化严重**——竞品分析有人做、广告视频也有人刷，但好的创意规律很难从竞品中提炼成自己可用的框架。
> 本文档记录完整 AI 介入方案，从创意提取到自动投放。

---

## 根本问题诊断

**表象**：想不出好创意，总是和别人拍得差不多
**根因**：竞品好作品的规律没有被结构化——只停留在"这个视频好"，没有转化成"前3秒用了什么钩子、情绪点是什么、产品卖点怎么呈现"

> 竞品分析员看了很多好视频，但输出的是一份文档 + 视频链接，而不是**可复用的创意公式**。

---

## 第一阶段：创意智能化（解决同质化）

### 核心逻辑

把竞品好广告拆解为**结构化标签**，积累成创意数据库，每次出创意从数据库里组合而非凭感觉想。

### 执行方法

**① 竞品视频结构化拆解**

对每条点击率高的竞品广告，提取以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 前3秒钩子类型 | 悬念/痛点/对比/场景 | "走了一天路脚还不疼？" |
| 情绪触发点 | 愉悦/痛苦/认同/好奇 | 认同（上班族久站） |
| 产品卖点呈现方式 | 穿着示范/特写细节/用前用后 | 鞋底特写+脚部放大 |
| 视觉风格 | 色调/节奏/字幕风格 | 暖色/快切/大字幕 |
| 结尾 CTA | 引导动作 | "点击购物车看尺码" |
| 估算效果 | 点赞率/评论情绪 | 高点击，评论多问尺码 |

**② 用 Claude 自动拆解**

将竞品视频前3秒截图 + 字幕文本发给 Claude，批量输出结构化标签，写入 Obsidian 创意数据库。

**③ 创意公式组合**

根据数据库里高频有效的组合，生成新创意 Brief：
```
钩子：痛点型（久站脚疼）
情绪：认同 → 解脱
产品呈现：穿着前后对比
视觉：暖色，节奏中等，字幕突出"8小时不累脚"
CTA：引导点评论问尺码
```

## 第二阶段：出图批量化

### 工具选型

| 需求 | 工具 | 说明 |
|------|------|------|
| 鞋类产品图生成 | Midjourney / FLUX | 多角度、多场景批量出图 |
| 姿态/角度控制 | ControlNet（姿态图） | 固定模特姿势，换鞋款 |
| 背景替换 | ComfyUI 工作流 | 产品图一键换背景（街头/办公/户外） |
| 批量执行 | ComfyUI API + Python | 100张图一次提交 |

### 流程

1. 上传产品主图（白底）
2. 选择场景模板（街头/通勤/运动）
3. ComfyUI API 批量生成带场景的产品图
4. 人工筛选 Top 20%，进入下一阶段

## 第三阶段：文案公式化

### 鞋类广告高转化文案结构

```
[痛点/场景]  →  [产品特性]  →  [结果/情绪]  →  [CTA]
```

**示例组合：**
- 上班族版："站了8小时，同事都在换鞋，我还没感觉 → 记忆棉中底 → 脚不疼才能工作到下班 → 评论问尺码"
- 时尚版："出门被问3次在哪买的 → 独家设计款 → 好看又舒服 → 点购物车"
- 对比版："换了这双之后再也不买其他品牌了 → 轻量大底 → 走路像踩云 → 限时价格"

**Claude 批量生成文案：**

给定产品卖点 + 目标人群 + 情绪方向，Claude 一次生成20条候选文案，团队选3-5条进入测试。

## 第四阶段：视频合成（出图→视频）

### 工具

| 环节 | 工具 |
|------|------|
| 图片转视频 | Kling AI / Runway Gen-3 |
| 配音/BGM | ElevenLabs（AI配音）+ 免版权BGM库 |
| 字幕合成 | CapCut API / FFmpeg |
| 批量渲染 | Python 脚本调用各 API |

### 自动化脚本逻辑

```python
for creative_brief in creative_list:
    # 1. 生成产品图（ComfyUI API）
    image = comfyui_generate(brief.product, brief.scene)
    # 2. 图片转视频（Kling API）
    video_clip = kling_image_to_video(image, motion="slow zoom")
    # 3. 生成配音（ElevenLabs）
    voiceover = elevenlabs_tts(brief.copy_text)
    # 4. 合成最终视频（FFmpeg）
    final_video = ffmpeg_merge(video_clip, voiceover, brief.caption)
    output_videos.append(final_video)
```

## 第五阶段：OpenClaw 自动投放与监控

### 核心原则：用官方 API，不用爬虫

> [!warning] 重要
> TikTok for Business 有官方的 **Marketing API**，支持程序化投放、出价调整、素材上传、数据拉取。
> **用官方 API = 零封号风险**，不需要绕过任何反爬机制。

### OpenClaw 多智能体架构

```
┌─────────────────────────────────────────────┐
│              Scheduler Agent                │
│  每天9点触发，分配任务给各子智能体            │
└──────────────────┬──────────────────────────┘
                   │
       ┌───────────┼───────────┐
       ▼           ▼           ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│ Creative │ │Production│ │Placement │
│  Agent   │ │  Agent   │ │  Agent   │
│竞品分析  │ │批量出图  │ │自动投放  │
│创意提炼  │ │视频合成  │ │出价调整  │
└──────────┘ └──────────┘ └──────────┘
                               │
                   ┌───────────┘
                   ▼
            ┌──────────┐
            │  Report  │
            │  Agent   │
            │数据汇总  │
            │Obsidian  │
            └──────────┘
```

### TikTok Marketing API 主要接口

| 功能 | API 端点 | 说明 |
|------|---------|------|
| 上传素材 | `/open_api/v1.3/file/video/ad/upload/` | 上传视频广告素材 |
| 创建广告组 | `/open_api/v1.3/adgroup/create/` | 设置预算、受众、版位 |
| 创建广告 | `/open_api/v1.3/ad/create/` | 绑定素材和落地页 |
| 拉取报表 | `/open_api/v1.3/report/integrated/get/` | 获取展示/点击/消耗数据 |
| 修改出价 | `/open_api/v1.3/adgroup/update/` | 动态调整 CPM/CPC 出价 |

### Report Agent 自动监控逻辑

```python
# 每小时拉一次数据
metrics = tiktok_api.get_report(ad_ids=active_ads, date_range="today")

for ad in metrics:
    # 烧钱过快：暂停
    if ad.spend_rate > budget_limit * 1.2:
        tiktok_api.pause_ad(ad.id)
        notify("⚠️ 广告 {} 烧钱过快，已暂停".format(ad.name))

    # CTR 低于阈值：降预算
    if ad.ctr < 0.8:
        tiktok_api.update_budget(ad.id, ad.budget * 0.7)

    # CTR 高表现：加预算
    if ad.ctr > 2.0 and ad.roas > 3.0:
        tiktok_api.update_budget(ad.id, ad.budget * 1.3)

# 每天下午6点生成日报写入Obsidian
generate_daily_report(metrics)
```

## 完整工作流总结

```
竞品视频/截图
     ↓
[Creative Agent] 结构化拆解 → 创意公式数据库
     ↓
[Production Agent] 出图 → 合成视频 → 批量素材包
     ↓
[Placement Agent] TikTok Marketing API 投放
     ↓
[Report Agent] 实时监控烧钱 → 自动调预算 → 日报到Obsidian
```

---

## 关键结论

1. **创意同质化的解法不是"更努力看视频"，而是建立结构化提炼机制**——把好视频翻译成可复用的公式
2. **TikTok 自动投放用官方 Marketing API**，合规、稳定、零封号风险，不需要任何反爬手段
3. **OpenClaw 的价值在于把4个阶段串联起来**——从竞品洞察到上线投放，减少人工中转，提升迭代速度
4. **出图环节的杠杆最大**：从手动 PS 改为 ComfyUI + ControlNet 批量生成，理论上可以把素材产出从"每周5条"提升到"每天50条"
