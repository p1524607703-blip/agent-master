# TikTok 广告 AI 全链路工作流程

> [!info] 文档说明
> 本文档回答：从零开始做一个 TikTok 鞋类广告视频，需要哪些 AI 工具、流程是怎样的、我们现在做到哪了。
> 基于 [[TikTok广告PRD-PRD总览-2026-03-29]] 及四个 Phase 子文档汇总。

---

## 一、完整 AI 工具链（4 Phase）

```
竞品分析员刷 TikTok
        │
        ▼
Phase 1 插件（Canvas + Claude Haiku）
拆解竞品视频 → 创意数据库
        │
        ▼
Phase 2 文案系统（Claude Sonnet）
输入产品 → 批量生成20条 → 策划筛选3-5条
        │
        ▼
Phase 3A 出图（ComfyUI + Claude Vision质检）
批量出图 → 人工抽检 → Kling 图生视频
Phase 3B 合成（Kling + ElevenLabs + Remotion + FFmpeg）
成品 MP4 + 溯源标签
        │
        ▼
Phase 4 投放（TikTok Marketing API + OpenClaw）
Smart+ 全自动投放 → ROAS 监控 → 飞书告警 → 数据回流
        │
        ▼
创意数据库更新（飞轮闭环）
```

---

## 二、各 Phase 工具详解

### Phase 1：竞品创意拆解

**工具**：Chrome 插件 + **Claude Haiku**（多模态图片分析）

| 组件 | 工具 | 说明 |
|------|------|------|
| 截帧 | Canvas API | 浏览器原生，无 DRM 问题 |
| 图片分析 | Claude Haiku | 结构化标签输出 |
| 本地存储 | IndexedDB | 三个 Store 持久化 |
| UI 隔离 | Shadow DOM | 完全不受 TikTok CSS 影响 |

**Claude 分析输出字段**：
- 钩子类型（痛点激发 / 好奇悬念 / 社交证明 / 前后对比 / 直接产品）
- 情绪弧线（起始 → 中段 → 结尾）
- 视觉风格（暖色/冷色 + 快切/慢节奏 + 大字幕/小字幕）
- CTA 类型（软引导 / 硬引导 / 无CTA）
- 一句话洞察

**→ 详情见**：[[TikTok广告PRD-Phase1-创意拆解插件]]

---

### Phase 2：文案撰写

**工具**：**Claude Sonnet 4-6**（创意写作，质量优先于 Haiku）

| 场景 | 模型 | 原因 |
|------|------|------|
| 文案生成 | Claude Sonnet 4-6 | 创意写作能力强 |
| 图片质检 | Claude Haiku | 速度快，成本低 |

**核心机制**：

- 批量生成 20 条候选文案
- 文案策划从中选 3-5 条（采用 / 淘汰 / 修改后采用）
- 采用率自动统计 → Prompt 版本质量追踪
- **冷启动策略**：数据库 < 50 条时使用行业通用公式模板兜底

**输出**：`ImageGenerationBrief` JSON（钩子句 + 正文 + CTA + 出图参数推导）

**→ 详情见**：[[TikTok广告PRD-Phase2-文案撰写系统]]

### Phase 3：素材生产

分两个子阶段，出图和视频合成串联。

#### 子阶段 A — 出图

**工具**：**ComfyUI**（主力）+ Claude Vision（质检）

| Sprint | 阶段 | 方式 |
|--------|------|------|
| Sprint 1（Week 4）| 调参验证 | ComfyUI Web UI 手动操作 |
| Sprint 2（Week 5+）| 自动化批量 | ComfyUI API + Fabric.js |

**出图成本估算**（Replicate API）：

```
50张/天 × $0.08/张 = $4/天 ≈ ¥29/天 ≈ ¥870/月
```

**UGC 风格防 TikTok 审核标记**（必须加入 Prompt）：

```
正向：shot on iPhone, casual lifestyle photography, natural lighting,
      authentic, slightly imperfect, real person wearing shoes

负向：cinematic, dramatic lighting, professional photography,
      CGI, rendered, 8k ultra detail, stock photo look
```

**质检流程（强制卡点）**：

1. AI 初筛：Claude Vision 检测脚趾变形 / 鞋底比例失调 / 明显合成痕迹
2. 人工抽检：每批次抽检 10%，批量勾选确认后进入视频合成
3. 拒绝记录：被拒图片原因回流到 ComfyUI 调参参考

**→ 详情见**：[[TikTok广告PRD-Phase3-素材生产]]

---

#### 子阶段 B — 视频合成

**工具链**：

| 工具 | 用途 | 成本 |
|------|------|------|
| **Kling AI** | 图生视频（主力） | ~$0.14/5秒视频 |
| **ElevenLabs** | AI 英文配音 | ~$0.30/1000字符 |
| **Remotion**（Node.js 服务） | 字幕逐字动效（服务端渲染） | 开源免费 |
| **FFmpeg** | 音视频合并 | 免费 |
| **TypeTale** | 仅人工精修兜底（无 API） | 桌面客户端 |

> [!warning] Remotion 架构约束
> Remotion 是 Node.js **服务端**渲染框架，不能在浏览器或 Chrome Extension 内运行。
> 必须有一个常驻 Node.js 服务（localhost:3001）接收渲染请求。

**TypeTale 使用场景**（明确，无 API，不能自动化）：

| 场景 | 是否用 TypeTale |
|------|---------------|
| 自动化视频合成（主流程） | ❌ 用 Kling + Remotion |
| AI 合成质量不达标时人工精修 | ✅ |
| 重点素材逐字动画精修 | ✅ |
| 风格提示词反推优化 ComfyUI | ✅ |

**合成成本估算**（20条/天）：

```
Kling：20 × $0.14 = $2.8
ElevenLabs：20 × $0.10 = $2.0
合计：~$4.8/天 ≈ ¥35/天
```

### Phase 4：投放与监控

**工具**：TikTok Marketing API + **OpenClaw Agent**

**核心原则**：出价/定向全交给 TikTok Smart+，我们只管素材和监控。

| 负责方 | 职责 |
|--------|------|
| TikTok Smart+ | 受众定向、出价策略、版位分配、素材轮换 |
| **Upload Agent（OpenClaw）** | 上传素材、绑定 Smart+ Campaign |
| **Report Agent（OpenClaw）** | ROAS 监控、异常告警（飞书）、日报推送 |

**Report Agent 告警规则**（少而准，宁漏报不误报）：

```
规则1（WARNING）：当天已花费 > 日预算×80%，距结束 > 6小时
规则2（ERROR）：花费 > $30 但转化为 0
规则3（ERROR）：花费 > $50 且 ROAS < 目标×0.5
```

**数据回流**：ROAS 数据写回创意数据库 → Phase 2 推荐越积累越准 → 完整飞轮

**→ 详情见**：[[TikTok广告PRD-Phase4-投放监控]]

## 五、完整流水线总结

> [!success] 一句话定位
> **Phase 1 发现** → **Phase 2 提炼** → **Phase 3 生产** → **Phase 4 投放** → **数据回流 Phase 1/2**
>
> 完整飞轮：创意数据库越积累 → 文案质量越高 → 素材转化越好

| 阶段 | 输入 | 输出 | 人工介入点 |
|------|------|------|-----------|
| Phase 1 创意拆解 | TikTok 竞品视频 | 结构化标签 JSON | 分析员校验标签 |
| Phase 2 文案撰写 | 创意数据库标签 | ImageGenerationBrief × 20条 | 策划选 3-5 条并评分 |
| Phase 3 素材生产 | 文案 Brief | 成品 MP4 素材包 | 出图后人工抽检图片 |
| Phase 4 投放监控 | 素材包 + 广告数据 | 自动调预算 + 日报 | 投手收飞书告警后决策 |
