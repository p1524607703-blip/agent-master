# TikTok 广告 AI 全自动化方案纲要

> [!warning] 核心问题
> 当前 Phase 1-4 方案的提效本质：**人找视频 → 人触发插件 → AI 分析 → 人做决策 → AI 生成**。
> 瓶颈在人这一端：竞品视频需要人去刷、去搜、去触发。
>
> 本文档探讨：**完全去掉人的触发环节，agent 自己完成从竞品发现到广告投放的全流程**。

---

## 二、当前方案中哪些环节可以 agent 化

### 2.1 各 Phase 自动化可能性评估

| Phase | 当前方案 | 可 agent 化程度 | 关键障碍 |
|-------|---------|----------------|---------|
| Phase 1 创意拆解 | 人刷 TikTok，插件截帧分析 | **可完全 agent 化** | TikTok 反爬机制（IP/行为检测） |
| Phase 2 文案撰写 | 人看分析结果，写 prompt | **可完全 agent 化** | 创意风格一致性（prompt 工程） |
| Phase 3 素材生产 | 人选图/视频片段，投喂 AI | **可完全 agent 化** | AI 生图一致性（品牌一致性） |
| Phase 4 投放监控 | 人看数据，决策调价 | **可完全 agent 化（TikTok Smart+）** | 预算上限决策（需人授权） |

**结论**：技术上 Phase 1-4 全部可以 agent 化。唯一障碍是 **Phase 1 的竞品视频发现**——TikTok 是封闭平台，不能像网页爬虫那样批量抓取。

## 三、Phase 1 的全自动化路径：三种方案

### 方案 A：TikTok 官方数据分析平台（推荐）

**思路**：不用爬虫，用 TikTok 官方数据源。

| 数据源 | 说明 | 可用性 |
|--------|------|--------|
| TikTok Business Center | 官方品牌数据分析 | 需要品牌账号，功能有限 |
| TikTok Creative Center | 热门广告创意榜单 | **✅ 可用**，无需登录 |
| TikTok Analytics API | 创作者数据分析 | 需要白名单 |
| TikTok Ads Manager | 广告数据 | **✅ 可用**，已投放才有数据 |

**TikTok Creative Center**（creativecenter.tiktok.com）是 TikTok 官方热门广告库，可以按行业/地区/时间筛选，合法合规，数据结构化。

```
方案 A 流程：
Agent 定时（每天/每周）
    ↓
访问 TikTok Creative Center
    ↓
按关键词（shoes/sneakers/running shoes）筛选
    ↓
抓取广告列表（标题、播放量、互动数据、落地页）
    ↓
截帧封面图 → Claude 多模态分析（无需进视频页）
    ↓
结构化数据写入数据库
    ↓
触发 Phase 2 文案生成
```

**优点**：
- 官方数据，完全合规，无封号风险
- 数据量大（按品类/地区/时间多维度）
- 结构化数据，比 DOM 抓取稳定

**缺点**：
- Creative Center 页面为 JS 渲染，需无头浏览器（Puppeteer/Playwright）
- 部分广告数据需登录才能查看完整指标

---

### 方案 B：竞品账号监控（替代方案）

**思路**：不随机抓取，改为监控已知竞品账号。

```
已知竞品账号列表（提前录入）
    ↓
TikTok 账号本身有公开 RSS/页面
    ↓
Puppeteer 无头浏览器定期访问竞品主页
    ↓
抓取最新 N 条视频列表
    ↓
截帧封面 + 基础数据
    ↓
Claude 分析 → Phase 2
```

**优点**：账号固定，数据源稳定，可精准追踪
**缺点**：需要人工先维护一份竞品账号列表（但这是最小化人工）

---

### 方案 C：浏览器插件 + Cloudflare Workers 中转（当前方案升级）

**思路**：保留 Phase 1 插件（人触发），同时增加 agent 后端。

```
人操作插件（当前）
    ↓
抓取数据存入 IndexedDB
    ↓
数据同步到云端（Cloudflare Workers + KV）
    ↓
Agent 读取云端数据 → 自动触发 Phase 2-4
    ↓
投放结果回写到数据库 → 形成反馈闭环
```

**优点**：人机协作，AI + 人工判断结合，质量更可控
**缺点**：仍依赖人的初始触发

### 4.3 Analysis Agent 详细设计

```python
# Analysis Agent — 创意结构化分析
# 触发方式：Discovery Agent 写入新数据后，或每小时轮询

class AnalysisAgent:
    def __init__(self):
        self.db = Database('creative_formula.db')
        self.claude = ClaudeHaikuClient()

    def batch_analyze(self, limit=50):
        pending = self.db.get_pending_analyses(limit=limit)

        for video in pending:
            frames = self.download_frames(video['cover_url'])  # 封面图 URL

            result = self.claude.analyze_frames(
                frames=frames,
                meta={'title': video['title'], 'keyword': video['keyword']}
            )

            # 写入分析结果
            self.db.update_analysis(video['id'], result)

            # 高分视频（estimated_engagement == '高'）→ 标记为候选
            if result['estimated_engagement'] == '高':
                self.db.mark_as_candidate(video['id'])

    def trigger_downstream(self):
        """分析完成后，自动触发文案生成 Agent"""
        candidates = self.db.get_candidates()
        if len(candidates) > 0:
            # 发布事件到消息队列（Redis/BullMQ）
            publish_event('HIGH_QUALITY_CREATIVE_FOUND', candidates)
```

### 4.4 Copywriting Agent 详细设计

```python
# Copywriting Agent — 批量文案生成
# 触发方式：Analysis Agent 发现高分创意，或每日定时

class CopywritingAgent:
    def __init__(self):
        self.db = Database('creative_formula.db')
        self.claude = ClaudeSonnetClient()

    def generate_batch(self, candidates, variant_count=3):
        """
        输入：候选创意列表（来自 Analysis Agent）
        输出：为每个创意生成 N 个文案变体

        variant_count=3：头部文案（高点击率导向）/ 腰部（情感共鸣）/ 测试型（新品强推）
        """

        generated = []

        for candidate in candidates:
            # 读取创意分析结果（钩子类型、情绪弧线、CTA类型）
            analysis = self.db.get_analysis(candidate['id'])

            for i in range(variant_count):
                prompt = self.build_copywriting_prompt(analysis, variant_type=i)
                copy = self.claude.generate(prompt)

                # 质量评分
                quality_score = self.claude.score_copy(copy)

                if quality_score >= 0.7:  # 质量阈值
                    self.db.save_copy_variant({
                        'source_video_id': candidate['id'],
                        'variant_type': ['head', 'emotional', 'test'][i],
                        'copy_text': copy['text'],
                        'hook_line': copy['hook_line'],
                        'cta': copy['cta'],
                        'quality_score': quality_score
                    })

                    # 触发图片生成 Brief
                    self.trigger_image_brief(copy, candidate)
```

### 4.5 Production Agent 详细设计

```python
# Production Agent — 素材生产（图片 + 视频）
# 触发方式：Copywriting Agent 生成文案后

class ProductionAgent:
    def __init__(self):
        self.db = Database('creative_formula.db')
        self.comfyui = ComfyUIClient()
        self.kling = KlingAIClient()

    def produce_material(self, copy_variant):
        # Step 1: 根据文案 Brief 生成图片
        brief = self.build_image_brief(copy_variant)
        image_job = self.comfyui.queue_image(brief)

        # 等待 ComfyUI 完成（WebSocket 回调，或轮询）
        image_result = self.wait_for_comfyui(image_job.job_id)

        # Step 2: 图片生视频（Kling AI）
        video_job = self.kling.img2video(
            image=image_result['image_url'],
            prompt=brief['motion_description']
        )

        video_result = self.wait_for_kling(video_job.job_id)

        # Step 3: 上传素材到 TikTok Ads
        uploaded = self.tiktok.upload_video(video_result['video_path'])

        # Step 4: 保存素材记录
        self.db.save_material({
            'copy_variant_id': copy_variant['id'],
            'image_url': image_result['image_url'],
            'video_url': uploaded['video_id'],
            'status': 'ready_for_deploy'
        })

        # Step 5: 触发 Deploy Agent
        self.trigger_deploy(uploaded)
```

### 4.6 Deploy + Monitor Agent 详细设计

```python
# Deploy Agent — 自动投放
# 触发方式：Production Agent 产出新素材

class DeployAgent:
    def __init__(self):
        self.db = Database('creative_formula.db')
        self.tiktok = TikTokAdsClient()

    def auto_deploy(self, material, budget_ceiling=50):
        """
        使用 TikTok Smart+（SPC）全自动投放
        Agent 仅执行，不决策预算
        """

        ad = self.tiktok.create_smart_plus_campaign(
            video_id=material['video_url'],
            ad_text=material['copy_text'],
            objective='CONVERSIONS',  # 转化
            daily_budget=budget_ceiling
        )

        self.db.save_deployment({
            'material_id': material['id'],
            'campaign_id': ad['campaign_id'],
            'status': 'running',
            'deployed_at': datetime.now()
        })

# Monitor Agent — 投放监控
class MonitorAgent:
    def run(self):
        """每小时轮询 TikTok Ads API"""
        running_campaigns = self.db.get_running_campaigns()

        for campaign in running_campaigns:
            metrics = self.tiktok.get_campaign_metrics(campaign['campaign_id'])

            # 规则 1：日消耗接近预算上限
            if metrics['daily_spend'] >= campaign['daily_budget'] * 0.95:
                self.feishu.alert_budget_exhausted(campaign)

            # 规则 2：高消耗零转化
            if metrics['spend'] > 30 and metrics['conversions'] == 0:
                self.tiktok.pause_campaign(campaign['campaign_id'])
                self.feishu.alert_zero_conversion(campaign)

            # 规则 3：ROAS 低于目标 50%
            if metrics['roas'] < campaign['target_roas'] * 0.5:
                self.tiktok.pause_campaign(campaign['campaign_id'])
                self.feishu.alert_low_roas(campaign)

            # 规则 4：高质量Campaign（ROAS > 2x 目标）→ 自动申请加预算
            if metrics['roas'] > campaign['target_roas'] * 2:
                self.feishu.request_budget_increase(campaign)  # 飞书通知人审批
```

## 六、Human Gate：人必须保留的 5 个决策节点

> [!warning] 全自动不是无人化
> 以下 5 个节点必须由人授权或审核，Agent 无权自行决定：

| # | 决策节点 | 决策内容 | Agent 权限 | 人介入方式 |
|---|---------|---------|-----------|-----------|
| 1 | **预算授权** | 新 Campaign 的日预算金额 | Agent 建议，人审批 | 飞书消息卡片 → [同意] [修改预算] |
| 2 | **品类扩展** | 新增关键词/竞品账号 | Agent 建议，人审批 | 每周策略审核会议 |
| 3 | **素材质量** | AI 生图/视频首次使用 | Agent 产出，人抽检 10% | 每日/每周抽检 |
| 4 | **危机处理** | 政策违规/账号异常/预算暴增 | Agent 告警，人处理 | 飞书紧急通知 |
| 5 | **策略方向** | 月度/季度广告策略 | 人决策 | 季度业务会议 |

**其余所有操作均为 Agent 全自动执行。**

## 七、技术架构：选择 Server 还是 Browser Extension

```
两种部署路径对比：

┌─────────────────────────────────┐    ┌─────────────────────────────────┐
│  路径 1：Server-only（云端 Agent） │    │  路径 2：Browser Extension + Server │
├─────────────────────────────────┤    ├─────────────────────────────────┤
│                                  │    │                                  │
│  Discovery Agent ──► 云端定时执行  │    │  Phase 1 插件（人触发）          │
│  用 Playwright/Puppeteer         │    │    ↓ 同步到云端                  │
│  抓取 Creative Center            │    │  Cloudflare Workers              │
│                                  │    │    ↓                            │
│  Analysis Agent ──► Claude API   │    │  Discovery Agent（定时补数）      │
│  Copywriting Agent               │    │    ↓                            │
│  Production Agent ──► ComfyUI API│    │  其余 Agent 同左                │
│                                  │    │                                  │
│  ✅ 真正的无人值守（24h 运行）      │    │  ✅ Phase 1 数据更真实（人筛选）   │
│  ✅ 不依赖用户电脑开机             │    │  ✅ 人机协作，质量可控            │
│  ❌ TikTok 反爬风险（需代理池）    │    │  ❌ 仍依赖人的初始触发            │
│  ❌ 维护成本高（服务器、监控）      │    │  ❌ Discovery 主动性弱            │
│                                  │    │                                  │
└─────────────────────────────────┘    └─────────────────────────────────┘

推荐路径：路径 1（Server-only）
理由：Phase 1 的目的是积累竞品数据库，只要数据能进库，
     路径是人触发还是 Agent 触发，最终效果一样。
     Server-only 的 24h 无人值守价值远大于维护成本。
```

**推荐基础设施**：
- 计算：阿里云 ECS / 香港轻量服务器（TikTok 主要市场）

- 代理：住宅代理池（TikTok 反爬必须）
- 部署：Docker Compose 单机运行，不引入 Kubernetes 复杂度
