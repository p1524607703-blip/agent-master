# TikTok 创意拆解插件 — UI 设计规范

> [!info] 文档说明
> 本文档为 [[TikTok广告PRD-Phase1-创意拆解插件-2026-03-29]] 的配套 UI 设计规范，面向前端开发者，描述所有 UI 组件、状态机、线框图和样式规范。

## 一、组件清单

| # | 组件名称 | 注入位置 / 页面 | 触发条件 | 尺寸规格 |
|---|---------|--------------|---------|---------|
| 1 | **悬浮触发按钮（视频页）** | `tiktok_video.js` 注入至 `document.body`，定位于页面左侧垂直居中 | 所有 `tiktok.com/*` 页面加载完成后常驻显示 | 48×48 px，含徽章时扩展至右上角 16×16 px |
| 2 | **悬浮触发按钮（搜索页高亮态）** | 同上，URL 匹配 `/search` 后切换为高亮状态 | `location.href` 包含 `/search` 时自动切换 | 48×48 px，发光描边 2px |
| 3 | **侧边主面板** | `tiktok_video.js` 注入至 `document.body`，定位页面右侧 | 点击悬浮按钮展开 | 宽 360 px，高 100% viewport，固定定位 |
| 4 | **模式A分析结果卡片** | 侧边主面板内容区 | 模式A分析成功后替换面板内容区 | 宽 100%（面板内），高 auto，最小 320 px |
| 5 | **"加入列表"询问栏** | 模式A分析结果卡片底部固定区域 | 分析卡片展示完成后立即显示，3秒倒计时 | 高 56 px，宽 100% |
| 6 | **模式B搜索感知气泡** | `tiktok_search.js` 注入至 `document.body`，定位悬浮按钮右侧 | URL 变为 `/search/...` 后 500ms 自动弹出 | 宽 320 px，高 auto（约 220 px） |
| 7 | **模式B视频列表预览** | 侧边主面板内容区 | 点击气泡"开始抓取"，元数据加载完成后显示 | 宽 100%（面板内），高 auto，可滚动 |
| 8 | **模式B批量分析进度面板** | 侧边主面板内容区 | 用户确认列表后点击"确认，开始 AI 分析" | 宽 100%（面板内），高 180 px |
| 9 | **收藏列表区域** | 侧边主面板底部固定区域 | 常驻，列表为空时折叠显示占位文案 | 高 auto，最大 240 px，超出滚动 |
| 10 | **导出操作栏** | 收藏列表区域下方 | 收藏列表有数据时显示 | 高 48 px，宽 100% |
| 11 | **Toast 通知条** | `document.body` 顶层，页面顶部居中 | 操作成功/失败时弹出 | 宽 280 px，高 40 px |
| 12 | **错误/降级提示卡** | 侧边主面板内容区 | API 失败/截帧失败等异常触发 | 宽 100%（面板内），高 auto |

### 2.2 悬浮按钮状态机

| 状态名 | 触发条件 | 视觉表现 |
|-------|---------|---------|
| `IDLE` | 在视频页，无任何操作 | 半透明白色圆形，图标正常显示 |
| `IDLE_WITH_BADGE` | IDLE + 收藏列表 ≥1 条 | 右上角显示绿色数字徽章 |
| `SEARCH_DETECTED` | URL 含 `/search` | 按钮变为品牌主色，脉冲发光动画 |
| `LOADING` | 分析进行中 | 旋转加载圈替代图标 |
| `PANEL_OPEN` | 面板已展开 | 高亮描边，图标变为"×" |

### 2.3 模式A完整状态机

```
IDLE
  │ 点击"分析当前视频"
  ▼
CAPTURING（截帧中）
  │ Canvas 截帧完成（约 1s）
  │ 失败 → ERROR_CAPTURE
  ▼
ANALYZING（API 请求中）
  │ 约 10-15 秒
  │ 超时/失败 → ERROR_API
  ▼
RESULT_CARD（展示分析结果）
  │ 同时展示底部询问栏，3秒倒计时
  ├─ 用户点击"＋ 加入列表" → LIST_ADDED
  ├─ 用户点击"跳过"        → RESULT_ONLY（无徽章更新）
  └─ 3秒自动关闭           → RESULT_ONLY（无徽章更新）

LIST_ADDED
  │ 写入 IndexedDB + 悬浮按钮徽章数字 +1
  ▼
RESULT_ONLY（结果卡片保留，可滚动查看）
```

### 2.4 模式B完整状态机

```
SEARCH_DETECTED（URL 含 /search）
  │ 500ms 延迟后
  ▼
BUBBLE_SHOWN（感知气泡弹出）
  ├─ 用户点击"忽略" → BUBBLE_DISMISSED（气泡收起，按钮恢复 IDLE）
  └─ 用户点击"开始抓取" →
       ▼
SCROLLING（自动滚动加载更多）
  │ 达到目标数量（N×2）或 maxScrolls=15
  ▼
DOM_EXTRACTING（提取视频卡片元数据）
  │ 排序方式 = 播放量 → VIEW_COUNT_FETCHING（逐条进入视频页）
  │ 其他排序方式 → 直接跳到 LIST_PREVIEW
  ▼
LIST_PREVIEW（视频列表预览，可勾选）
  ├─ 用户调整勾选后
  └─ 点击"确认，开始 AI 分析" →
       ▼
BATCH_ANALYZING（逐条截帧+API，显示进度）
  │ 用户点击"暂停" → BATCH_PAUSED（可恢复）
  │ 全部完成 →
  ▼
BATCH_DONE（合并去重写入 IndexedDB）
  │
  ▼
UNIFIED_LIST（统一列表更新，面板切换至列表视图）
```

### 2.5 统一列表状态机

| 状态 | 触发 | 表现 |
|-----|------|-----|
| `LIST_EMPTY` | 初始状态，无数据 | 显示占位文案"尚无分析记录" |
| `LIST_HAS_ITEMS` | 有 ≥1 条数据 | 显示列表条目 + 导出操作栏 |
| `LIST_EXPORTING_CSV` | 点击"导出 CSV" | 按钮 loading 态，完成后 Toast 提示 |
| `LIST_PUSHING_FEISHU` | 点击"推送飞书" | 按钮 loading 态，成功/失败各有 Toast |

### 4.4 Shadow DOM 隔离策略

#### 原则：完全隔离，零依赖 TikTok 样式

```javascript
// 注入方式（content_scripts/tiktok_video.js）

function injectUI() {
  // 1. 创建宿主元素，使用唯一 ID 防止重复注入
  const HOST_ID = 'tiktok-creative-ext-root';
  if (document.getElementById(HOST_ID)) return;

  const hostEl = document.createElement('div');
  hostEl.id = HOST_ID;

  // 2. 定位样式直接写在宿主元素 style 上（不进入 Shadow DOM）
  //    避免 TikTok 样式重置影响宿主元素定位
  Object.assign(hostEl.style, {
    position: 'fixed',
    top: '0',
    left: '0',
    width: '0',
    height: '0',
    zIndex: '2147483647',  // 最大 z-index
    pointerEvents: 'none', // 宿主不捕获事件
    overflow: 'visible',
  });

  document.body.appendChild(hostEl);

  // 3. 创建 Shadow Root（closed 模式，外部无法访问）
  const shadow = hostEl.attachShadow({ mode: 'closed' });

  // 4. 将插件 CSS 注入 Shadow Root（用 <style> 标签，不用外链）
  const style = document.createElement('style');
  style.textContent = SHADOW_CSS; // 打包时内联
  shadow.appendChild(style);

  // 5. 创建实际 UI 容器
  const container = document.createElement('div');
  container.className = 'ext-root';
  shadow.appendChild(container);

  // 6. 渲染 UI 到 container
  renderApp(container);
}
```

#### 隔离要点清单

| 风险点 | 处理方案 |
|-------|---------|
| TikTok 全局 CSS Reset 影响插件样式 | Shadow DOM 天然隔离，外部样式不穿透 |
| TikTok `* { box-sizing: border-box }` 类规则 | Shadow Root 内部自定义 `box-sizing`，不受外部影响 |
| TikTok 修改 `body` 的 `overflow` 导致面板滚动失效 | 面板使用 `position: fixed` + 独立 `overflow-y: auto` |
| 插件 z-index 被 TikTok 模态框覆盖 | 宿主元素 `z-index: 2147483647`；必要时监听 TikTok DOM 变化后重新置顶 |
| TikTok 动态加载的 CSS 文件影响 Shadow DOM | Shadow DOM 内部的 CSS 完全自包含，外部新增样式无法穿透 |
| `:root` CSS 变量无法在 Shadow DOM 内使用 | 在 Shadow Root 的 `<style>` 顶部重新声明所有 CSS 变量 |
| font-size 被 TikTok 修改 `html { font-size: xx }` | Shadow Root 内根容器显式设置 `font-size: 14px`（绝对值） |
| TikTok 事件（如键盘快捷键）与插件冲突 | 插件事件监听器使用 `event.stopPropagation()`，仅在必要时 |

#### Shadow CSS 开头模板


### 5.1 视频页不同布局下悬浮按钮的位置适配

TikTok 在不同页面有三种主要布局，按钮定位策略如下：

| 布局场景 | URL 特征 | 按钮定位策略 | 注意点 |
|---------|---------|------------|-------|
| 沉浸式竖屏滚动页（FYP） | `tiktok.com/foryou` 或 `/` | `position: fixed; left: 8px; top: 50%` | 避开右侧 like/comment/share 按钮区域（右边 56px 宽） |
| 视频详情页（点开单条） | `/video/...` | 同上，`left: 8px` | 检查页面是否有左侧评论区抽屉，如有则改为 `left: 72px` |
| 搜索结果页 | `/search/...` | 同上，按钮切换为 SEARCH_DETECTED 态 | 搜索页有顶部 Tab 栏，按钮 `top` 避免被 Tab 覆盖，使用 `top: calc(50% + 32px)` |

**检测右侧按钮区域的代码逻辑：**
```javascript
// 若 TikTok 原生右侧按钮栏 DOM 出现，动态调整悬浮按钮位置
function adjustButtonPosition() {

                     || document.querySelector('.side-bar');

    const rect = nativeButtons.getBoundingClientRect();
    // 按钮始终在原生按钮区域左侧 8px
    floatBtn.style.left = Math.max(8, rect.left - 56) + 'px';
  }
}
```
