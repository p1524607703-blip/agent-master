# Phase 3 素材生产系统 — Web UI 设计

> [!info] 文档说明
> 本文档基于 [[TikTok广告PRD-Phase3-素材生产-2026-03-29]] PRD 产出，面向前端开发工程师。
> 包含：页面结构、组件设计、状态机、API 接口、Vue 3 组件拆分。
>
> **两类用户**：运营/设计师（操作层）+ 老板/管理层（数据层）
> **刷新方式**：前端轮询，每 30 秒刷新一次状态

---

## 一、页面结构与路由

```
路由设计（Vue Router）：

/phase3                       → 重定向到 /phase3/production
/phase3/production            → 【主视图】生产队列（运营/设计师）
/phase3/quality               → 【主视图】质检台（运营/设计师）
/phase3/video                 → 【主视图】视频合成进度（运营/设计师）
/phase3/library               → 【主视图】素材库（运营/设计师 + 管理层）
/phase3/dashboard             → 【主视图】数据看板（管理层）
/phase3/settings              → 【次要】ComfyUI / Kling API 配置
```

**权限分层**：
- `/production` `/quality` `/video` `/library` → 所有用户
- `/dashboard` → 默认展示综合数据看板，老板专用
- `/settings` → 仅管理员

## 三、API 接口设计

### 3.1 接口列表

| 方法 | 路径 | 说明 | 轮询频率 |
|------|------|------|---------|
| `GET` | `/api/phase3/tasks` | 获取所有生产任务及状态 | 30s |
| `GET` | `/api/phase3/tasks/:briefId` | 获取单个 Brief 详情 | 15s（任务进行中） |
| `GET` | `/api/phase3/quality-queue` | 获取待质检队列 | 30s |
| `POST` | `/api/phase3/quality-approve` | 批量确认质检结果 | 操作时触发 |
| `POST` | `/api/phase3/quality-reject` | 重新生成指定图片 | 操作时触发 |
| `GET` | `/api/phase3/video-progress/:briefId` | 视频合成进度 | 15s（合成中） |
| `GET` | `/api/phase3/library` | 素材库列表 | 30s |
| `GET` | `/api/phase3/stats` | 数据看板聚合数据 | 30s |
| `POST` | `/api/phase3/launch-to-phase4` | 推送素材到 Phase 4 | 操作时触发 |

### 3.2 核心数据结构

```typescript
// 任务状态枚举
type TaskStatus =
  | 'pending'           // 等待处理
  | 'generating_images'  // ComfyUI 出图中
  | 'awaiting_qc'        // 等待质检
  | 'video_synthesizing' // 视频合成中
  | 'completed'          // 全部完成，素材入库
  | 'failed';            // 失败

// 生产任务
interface ProductionTask {
  brief_id: string;
  source_brief_title: string;
  received_at: string;         // ISO 时间
  status: TaskStatus;
  image_generation: {
    total: number;
    done: number;
    passed_ai: number;
    flagged_ai: number;
    approved_human: number;
    current_node: string;     // 当前 ComfyUI 节点
    eta_seconds: number;
  } | null;
  video_synthesis: {
    total: number;
    done: number;
    current_video_index: number;
    eta_seconds: number;
    remotion_status: 'pending' | 'rendering' | 'done';
  } | null;
  material_id?: string;        // Phase 3 完成后，素材在库中的 ID
}

// 数据看板统计
interface ProductionStats {
  period: 'day' | 'week' | 'month';
  generated_total: number;
  image_total: number;
  video_total: number;
  quality_pass_rate: number;      // 合格率
  ai_agree_rate: number;           // AI 与人工判定一致率
  top_issue_reasons: string[];     // 最高频 AI 标记问题
  trend_data: { date: string; pass_rate: number }[];
  scene_distribution: { scene: string; count: number; rate: number }[];
}
```

## 五、轮询策略

```typescript
// composables/usePolling.ts
export function usePolling(fetchFn: () => Promise<void>, interval = 30000) {
  const isActive = ref(true);

  let timer: ReturnType<typeof setInterval>;

  async function poll() {
    if (!isActive.value) return;
    await fetchFn();
    timer = setTimeout(poll, interval);
  }

  // 任务状态变化时动态调整间隔
  function setFastMode(fast: boolean) {
    clearTimeout(timer);
    poll(); // 立即执行一次
    timer = setTimeout(poll, fast ? 15000 : 30000);
  }

  onMounted(() => poll());
  onUnmounted(() => {
    isActive.value = false;
    clearTimeout(timer);
  });

  return { setFastMode };
}
```
