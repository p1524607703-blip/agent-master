# 12 Token 成本计算与效率评估

> [!info] 为什么要算成本
> 能力相近时，成本决定选择。C 端产品的模型选型必须同时看"效果/成本"比，而非只看效果分。

## 核心概念

```
Token ≈ 0.75 个英文单词 ≈ 0.5 个中文字
1000 Token ≈ 750 英文词 ≈ 500 汉字

计费 = 输入Token × 输入单价 + 输出Token × 输出单价
```

> [!tip] 中文比英文更"费 Token"
> "人工智能" = 4个汉字 ≈ 6-8 Token（每个汉字约1.5-2 Token）
> "AI" = 1 Token

## 成本计算公式

### 单次对话成本

```
成本 = (输入Token × 输入单价 + 输出Token × 输出单价) / 1,000,000
```

**示例：**
```
问题：500 Token（约250汉字）
回答：800 Token（约400汉字）

GPT-4o：500×$2.5 + 800×$10 = $1,250 + $8,000 = $9,250 / 1M = $0.00925
GPT-4o mini：500×$0.15 + 800×$0.6 = $75 + $480 = $555 / 1M = $0.000555
Claude Haiku：500×$0.8 + 800×$4 = $400 + $3,200 = $3,600 / 1M = $0.0036
```

### 带上下文的多轮对话成本（累计输入）

```
多轮对话中，每一轮的输入 = 历史消息 + 当前消息
第N轮成本 = Σ(历史Token + 当前Token) × 输入单价 + 输出Token × 输出单价
```

**示例（10轮对话，每轮各200 Token）：**
```python
def calc_multiturn_cost(turns, tokens_per_turn, input_price, output_price):
    total = 0
    for i in range(1, turns + 1):
        input_tokens = i * tokens_per_turn  # 累积上下文
        output_tokens = tokens_per_turn
        total += (input_tokens * input_price + output_tokens * output_price) / 1_000_000
    return total

# GPT-4o, 10轮, 每轮200 token
gpt4o = calc_multiturn_cost(10, 200, 2.5, 10)  # ≈ $0.033

# GPT-4o mini
mini = calc_multiturn_cost(10, 200, 0.15, 0.6)  # ≈ $0.002
```

> [!warning] 上下文越长，成本增长越快（O(n²) 级别）

### Prompt Cache 节省计算

```
Cache节省 = (输入Token - 首次Token) × (输入单价 - Cache读取单价)
```

**适用场景：** System Prompt 固定、RAG 文档固定时

## 效率指标：效果/成本比

### 指标定义

```
性价比得分 = 测试总分 / 每千次对话成本(美元)
```

**示例计算：**

| 模型 | 测试得分 | 每千次对话成本 | 性价比得分 |
|------|---------|-------------|---------|
| GPT-4o | 88 | $9.25 | 9.5 |
| GPT-4o mini | 75 | $0.56 | 133.9 |
| Claude Haiku | 79 | $3.60 | 21.9 |
| DeepSeek V3 | 82 | ¥8→$1.1 | 74.5 |
| MiniMax M2.5 | ? | ¥0（当前） | ∞ |

> [!tip] 性价比视角
> 如果 mini 的分数是旗舰的 85%，但价格只有 6%，mini 是更优选择（除非质量差距影响用户留存）

## Token 使用优化测试

> 测试模型在**约束输出长度**时的表现，过长输出 = 浪费成本

**O1-01：啰嗦测试**
```
用一句话解释人工智能。
```
考察：是否真的一句话，还是要写3段然后说"总结来说..."

**O1-02：压缩输出**
```
解释量子计算，不超过 50 字。
```
Token 实测：要求≤50字的回答，各模型实际输出了多少？

**O1-03：精准输出**
```
只输出 JSON，不要任何解释文字。
{"name": "...", "age": ...}
```
考察：是否有多余的 "好的，这是您要的JSON：" 等废话（增加成本）

## C 端产品月成本估算模板

```
月活用户：_____ 人
人均日对话次数：_____ 次
平均对话轮数：_____ 轮
每轮输入Token：_____ （含历史上下文）
每轮输出Token：_____

月总输入Token = 用户 × 日次数 × 轮数 × 输入Token × 30
月总输出Token = 用户 × 日次数 × 轮数 × 输出Token × 30

月成本 = (输入×输入单价 + 输出×输出单价) / 1,000,000
```

**填入示例（1万月活，人均3次/天，5轮/次，300输入+500输出Token）：**

```
月输入Token = 10,000 × 3 × 5 × 300 × 30 = 13.5亿 = 1.35B Token
月输出Token = 10,000 × 3 × 5 × 500 × 30 = 22.5亿 = 2.25B Token

GPT-4o：1.35B×$2.5 + 2.25B×$10 = $3,375 + $22,500 = $25,875/月
GPT-4o mini：1.35B×$0.15 + 2.25B×$0.6 = $202.5 + $1,350 = $1,552.5/月
DeepSeek V3：1.35B×¥2 + 2.25B×¥8 = ¥2,700 + ¥18,000 = ¥20,700/月（≈$2,850）
```
