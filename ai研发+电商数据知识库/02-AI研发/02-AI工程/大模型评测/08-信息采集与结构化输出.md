# 08 最新信息采集与结构化输出

> [!info] 理论依据
> - **WebGPT（Nakano et al., 2021）**：带检索增强的问答，考察信息来源质量
> - **FRAMES（Google, 2024）**：多跳事实检索+综合能力
> - **ToolBench（Qin et al., 2023）**：工具调用正确性评测
> - **StructBench**：结构化格式生成质量

## 两个子维度说明

```
本节测试
├── A. 信息采集能力（有联网/RAG工具时）
│   ├── 能否识别"需要查新信息"的请求
│   ├── 查到的信息是否准确、是否引用来源
│   └── 知识截止日期边界的诚实性
└── B. 结构化输出（无论是否联网）
    ├── JSON / YAML / Markdown表格
    ├── 严格 Schema 遵从
    └── 嵌套结构与数据类型正确性
```

---

## A. 信息采集能力测试

### 【时效性识别】

**I1-01**
```
特斯拉现在的股价是多少？
```
**无联网时**预期：明确说明无实时数据，给出知识截止日期，建议去哪里查
**有联网时**预期：调用搜索工具，返回带时间戳的数据，注明来源

**I1-02**
```
最近有什么关于 AI 大模型的重大新闻？
```
**无联网时**预期：说明信息可能过时，列出知识截止前的重要事件
**有联网时**预期：检索近期新闻，多来源综合，注明检索时间

**I1-03（主动识别需联网）**
```
帮我分析一下现在买黄金合不合适。
```
考察：是否主动说明需要实时价格数据，而非基于过时信息给出"分析"

---

### 【RAG 信息质量】（适用于接入知识库场景）

**I2-01**
```
根据提供的文档，这个产品的退款政策是什么？
（文档中没有退款政策相关内容）
```
预期：说明文档中未找到相关信息，而非编造

**I2-02（多文档综合）**
```
把三份报告中关于"用户增长"的数据整合成一张表。
```
考察：跨文档信息对齐、数据来源标注

**I2-03（来源引用）**
```
用检索到的信息回答：为什么近期美联储选择降息？
```
评分标准：
- ✓ 引用具体来源（网页标题/日期）
- ✓ 区分"事实"和"分析"
- ✓ 信息互相印证而非单一来源

## B. 结构化输出测试

### 【JSON 严格模式】

**I3-01（基础）**
```
把以下用户信息转成 JSON，字段：name(string), age(integer),
tags(array of string), active(boolean):
姓名：李雷，年龄：25，标签：设计师、摄影爱好者，状态：活跃
```
评分：✓ 类型正确（age不加引号）✓ 布尔值用true/false ✓ 无多余字段

**I3-02（嵌套结构）**
```
将以下订单转成 JSON：
订单号 20240315-001，用户ID u789，
商品：手机壳(¥29, 2件)、钢化膜(¥15, 1件)，
收货地址：上海市浦东新区某路1号，邮编200120
```
考察：嵌套对象、数组、数值与字符串区分

**I3-03（Schema 遵从）**
```
严格按照以下 Schema 输出，不得添加额外字段：
Schema: { "status": "success"|"fail", "code": integer, "data": object|null }

场景：请求成功，返回用户信息 {id: 1, name: "测试"}
```

---

### 【Markdown 表格与报告】

**I4-01（数据表格）**
```
将以下销售数据转成 Markdown 表格，并在最后一行加合计：
1月：iPhone 300台，华为 200台
2月：iPhone 250台，华为 280台
3月：iPhone 320台，华为 190台
```

**I4-02（结构化报告）**
```
根据以下信息，输出一份结构化竞品分析报告，
格式：用 Markdown，包含概述、优势、劣势、机会、威胁四个部分，
每部分用要点列表：
[提供某产品信息...]
```

---

### 【格式稳健性测试】

**I5-01（长文本中保持格式）**
```
给我输出一份包含 10 个 API 接口的文档，每个接口包含：
接口名、URL、方法(GET/POST)、参数列表(JSON格式)、返回示例。
```
考察：10个接口全程格式一致，不在中途"漂移"

**I5-02（格式+内容双约束）**
```
以 JSON 格式输出 5 个适合初学者的 Python 练习题，
每题包含：title, difficulty(1-3), description, hint 字段
```

## 评分标准

### 信息采集（各题满分5分）

| 分数 | 标准 |
|------|------|
| 5 | 正确识别信息时效性，检索准确，来源清晰 |
| 4 | 基本正确，来源略显模糊 |
| 3 | 给出信息但未注明时效性或来源 |
| 2 | 混用过时知识和当前数据，未区分 |
| 1 | 直接编造"最新"数据 |

### 结构化输出（程序可验证）

```python
# 评分脚本示意
import json

def score_json_output(raw_output, expected_schema):
    try:
        parsed = json.loads(raw_output)        # +2: 可解析
        type_correct = check_types(parsed, expected_schema)  # +2: 类型正确
        no_extra_fields = check_fields(parsed, expected_schema)  # +1: 无多余字段
        return 2 + type_correct*2 + no_extra_fields
    except json.JSONDecodeError:
        return 0  # 不可解析直接0分
```
