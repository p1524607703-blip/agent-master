# 大模型 C 端能力测试框架

> [!abstract] 框架来源
> 综合 MT-Bench、Chatbot Arena、AlpacaEval、TruthfulQA、MMLU 等学术评测体系，结合 C 端用户真实使用场景设计。

## 为什么 C 端测试不同于学术 Benchmark

学术 Benchmark（MMLU、HellaSwag）测的是**模型能力上限**，但 C 端用户关心：
- 回答**有没有用**，不是选项对不对
- 表达**自不自然**，不是困惑度低不低
- 会不会**一本正经地胡说**（幻觉）
- 拒绝有没有**多余**，该帮的帮了没有

核心参考：**Chatbot Arena（LMSYS，2023）**的核心洞察：人类偏好 ≠ 学术分数。

## 评测方法论

### 方法一：人类偏好对比（主观）
参考 **Chatbot Arena** 的盲测对比法：
```
同一问题 → 两个模型匿名回答 → 用户选择更好的
```
> 优点：最接近真实体验；缺点：耗时，难以规模化

### 方法二：LLM-as-Judge（半自动）
参考 **MT-Bench（Zheng et al., 2023）**：用 GPT-4 作为评委打分（1-10分），评估：
- 有帮助性（Helpfulness）
- 相关性（Relevance）
- 准确性（Accuracy）
- 深度（Depth）
- 创造性（Creativity）
- 详细程度（Detail）

### 方法三：固定用例打分（半客观）
每个维度准备 10-20 道标准题，用统一 Rubric 评分。

## 快速开始

> [!tip] 最小可行测试集（30分钟版）
> 1. 指令遵循：5题（格式控制 + 角色扮演 + 拒绝遵从）
> 2. 幻觉检测：5题（陷阱事实题）
> 3. 多轮对话：2轮×3组
> 4. 安全红线：5题
> 5. 中文理解：5题
>
> **共约 25 题，用 LLM-as-Judge 自动评分，1小时出报告**
