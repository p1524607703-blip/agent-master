## 2. xiaomi:// URL 完整格式

### 基本格式

```
xiaomi://用户名:密码@摄像头IP[:端口]?参数1=值1&参数2=值2
```

### C700 完整配置示例

```yaml
# go2rtc.yaml
streams:
  c700:
    - xiaomi://xiaomi_account:password@192.168.1.100?did=设备DID&model=chuangmi.camera.81ac1
```

### URL 各部分说明

| 组成部分 | 说明 | 示例 |
|---------|------|------|
| `xiaomi://` | 协议前缀，go2rtc 内置的小米摄像头协议 | — |
| `用户名` | 小米/米家账号（手机号或邮箱） | `13800138000` |
| `密码` | 小米账号密码 | `MyPassword123` |
| `@摄像头IP` | 摄像头局域网 IP 地址 | `192.168.1.100` |
| `[:端口]` | 可选，默认由协议决定 | 通常省略 |

### 查询参数速查表

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `did` | 推荐 | 设备 ID（米家 App 中获取） | `did=1234567890` |
| `model` | 推荐 | 设备型号标识 | `model=chuangmi.camera.81ac1` |
| `token` | 可选 | 设备 Token（替代用户名密码认证） | `token=ffffffffffffffff...` |
| `subtype` | 可选 | 流质量选择（部分型号支持） | `subtype=3`（HD 质量） |

> [!tip] 获取设备 DID
> 在米家 App 中：设备 → 右上角设置 → 关于 → 设备信息中可找到设备 DID。
> 或通过 go2rtc Web UI（`http://localhost:1984`）登录米家账号后从设备列表获取。

## 3. CS2 协议说明

### 什么是 CS2

CS2 是小米（米家）摄像头的**私有流媒体协议**，用于摄像头与客户端之间的实时音视频传输。go2rtc 通过逆向工程实现了对该协议的支持。

### 传输方式

| 传输方式 | 说明 | 适用场景 |
|---------|------|---------|
| `cs2` | 默认 CS2 协议（自动选择传输层） | 大多数场景 |
| `cs2+udp` | CS2 协议 + UDP 传输 | C700 等较新型号使用此方式 |
| `cs2+tcp` | CS2 协议 + TCP 传输 | 部分型号（如 C200）使用 |

C700 使用的 `cs2+udp` 表示：
- 控制信令通过 CS2 协议建立连接
- 音视频数据通过 **UDP** 传输（低延迟，适合实时流）
- go2rtc 接收 UDP 数据后转码为标准 RTSP 流输出

### 与 RTSP 的关系

```
小米摄像头 --[cs2+udp]--> go2rtc --[RTSP]--> Home Assistant / VLC / OpenClaw
```

go2rtc 充当协议桥梁，将私有的 CS2 协议转换为通用的 RTSP 协议。

## 4. #2048 i/o timeout 回归问题

### 问题概述

| 项目 | 说明 |
|------|------|
| **Issue** | [#2048](https://github.com/AlexxIT/go2rtc/issues/2048)（Open） |
| **错误信息** | `read udp [::]:port: i/o timeout` |
| **影响版本** | v1.9.14 |
| **正常版本** | v1.9.13 |
| **影响协议** | CS2（UDP 和 TCP 传输均受影响） |
| **影响机型** | `chuangmi.camera.81ac1`（C700）、`xiaomi.camera.082ac1` 等 |

### 具体表现

1. go2rtc v1.9.14 启动后，摄像头流反复尝试连接
2. 约 5 秒后超时，报 `i/o timeout` 错误
3. 下游 RTSP 客户端收到 `404 Not Found`
4. 降级到 v1.9.13 后同一配置立即恢复正常

### 规避方案

> [!warning] 必须锁定 go2rtc v1.9.13

**Docker 部署时锁定版本：**
```bash
# 使用 v1.9.13 而非 latest
docker run -d \
  --name go2rtc \
  -p 1984:1984 \
  -v ./go2rtc.yaml:/config/go2rtc.yaml \
  alexxit/go2rtc:v1.9.13
```

**docker-compose.yaml：**
```yaml
services:
  go2rtc:
    image: alexxit/go2rtc:v1.9.13   # 锁定版本，不要用 latest
    ports:
      - "1984:1984"
      - "8554:8554"   # RTSP
      - "8555:8555"   # WebRTC
    volumes:
      - ./go2rtc.yaml:/config/go2rtc.yaml
    restart: unless-stopped
```

**注意事项：**
- v1.9.14 修复了米账号登录设备列表为空的问题（#2158），但引入了 CS2 超时回归
- 如果使用 v1.9.13 时遇到登录问题，先在浏览器登录米家完成验证再使用 go2rtc
- 持续关注 #2048 状态，等待官方修复后再升级

## 5. go2rtc 完整配置参考

### 最小配置（C700 专用）

```yaml
# go2rtc.yaml
streams:
  c700:
    - xiaomi://your_xiaomi_account:your_password@192.168.1.100?did=YOUR_DID&model=chuangmi.camera.81ac1
```

### 完整配置（含 API/RTSP/日志）

```yaml
# go2rtc.yaml — C700 完整配置
api:
  listen: ":1984"       # Web UI + HTTP API 端口

rtsp:
  listen: ":8554"       # RTSP 服务端口
  default_query: "video&audio"

webrtc:
  listen: ":8555"       # WebRTC 端口

log:
  level: "info"         # 调试时可改为 "debug" 或 "trace"

streams:
  c700:
    - xiaomi://your_account:your_password@192.168.1.100?did=YOUR_DID&model=chuangmi.camera.81ac1
```

### 端口说明

| 端口 | 协议 | 用途 |
|------|------|------|
| 1984 | HTTP | Web UI 管理界面 + REST API（截图、流列表） |
| 8554 | RTSP | RTSP 流输出，供 HA / VLC / FFmpeg 拉取 |
| 8555 | TCP/UDP | WebRTC 端口，供浏览器实时预览 |

### go2rtc 内置 API（非 HA）

| 接口 | 说明 |
|------|------|
| `GET http://go2rtc:1984/api/frame.jpeg?src=c700` | 直接从 go2rtc 获取截图 |
| `GET http://go2rtc:1984/api/streams` | 列出所有已配置的流 |
| `GET http://go2rtc:1984/` | Web UI 管理界面 |

## 6. 已知小米摄像头兼容表

> 数据来源：go2rtc [#1982](https://github.com/AlexxIT/go2rtc/issues/1982) Known Xiaomi Cameras
> 2020 年后生产的摄像头基本都能支持

### 已确认支持型号

| 产品名称 | 型号 ID | 协议 | 视频 | 音频 | 备注 |
|----------|---------|------|------|------|------|
| **小米 C700** | `chuangmi.camera.81ac1` | cs2+udp | H265 | OPUS | v1.9.14 受 #2048 影响 |
| Mi 360 2K Pro (2020) | `chuangmi.camera.021a04` | cs2 | H265 | PCMA | |
| Xiaomi Smart Camera 2 PTZ (2023) | `chuangmi.camera.039c01` | cs2+tcp | H265 | OPUS | |
| Xiaomi C200 (2023) | `chuangmi.camera.046c04` | cs2+tcp | H265 | OPUS | subtype=3 为 HD |
| Mi Camera 2K (2021) | `isa.camera.hlc7` | cs2+udp | H265 | PCMA | 默认 SD，H265 SPS 可能异常 |
| Xiaomi CW400 (2022) | `isa.camera.hlc8` | cs2 | H265 | OPUS | 室外型号 |

### 已知不支持型号

| 型号 ID | 原因 |
|---------|------|
| `chuangmi.camera.ipc017` | 官方标记 Not Planned（#2139） |
| `chuangmi.camera.v3` | 旧款（2017），待评估（#2171） |

### 旧款型号（2017-2019）

旧款摄像头使用 TUTK 协议，需要通过 `xiaomi/legacy` 或 `xiaomi/miss` 模式接入，兼容性有限。建议优先使用 2020 年后的型号。

## 7. 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `i/o timeout` | go2rtc v1.9.14 CS2 回归 | 降级到 v1.9.13 |
| 设备列表为空 | v1.9.13 的已知问题 | 先在浏览器登录米家，或升级到 v1.9.14 后换回 v1.9.13 |
| 滑块验证码无法通过 | 首次登录触发风控 | 先用浏览器在 mi.com 完成登录和验证 |
| 双向音频噪声 | go2rtc 音频编码问题（#2175） | 暂时关闭双向音频 |
| 流画面模糊 | 默认输出 SD 流 | 添加 `subtype=3` 参数切换 HD |
| RTSP 404 | 流名称不匹配 | 确认 go2rtc.yaml 中的 stream 名称 |

### 调试步骤

1. **检查 go2rtc 日志**：将 `log.level` 改为 `debug`
2. **访问 Web UI**：`http://localhost:1984/` 查看流状态
3. **测试 RTSP**：`ffplay rtsp://localhost:8554/c700`
4. **测试截图**：`curl http://localhost:1984/api/frame.jpeg?src=c700 -o test.jpg`
