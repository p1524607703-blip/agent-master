# Introduction to Structured Output Generation

> [!info] 课程信息
> **课程**：Getting Structured LLM Output
> **平台**：DeepLearning.AI

## 核心概念

- **结构化输出（Structured Output）**：让 LLM 返回有固定格式的内容（如 JSON），而不是随意的自然语言，是构建可扩展 AI 软件的基础
- **Logit 约束解码（Constrained Decoding）**：在模型生成 token 时直接过滤不合法 token，重新归一化概率，从根本上保证输出符合结构约束
- **Re-prompting 库**：当模型输出不合法时自动将错误原因追加到提示词重试，代表工具：Instructor、LangChain
- **函数调用 / 工具调用（Function Calling）**：让模型以 JSON 格式响应预定义函数，是主流 API 提供商的常见实现方式
- **从 Prompt Hacking 到 AI Engineering**：本课程的核心主张——结构化输出是让 LLM 从"聊天玩具"变成"工程组件"的关键一步

## 技术要点

**三种结构化输出方案对比：**

| 方案 | 代表 | 优点 | 缺点 |
|------|------|------|------|
| 厂商 API（JSON Mode / Function Calling） | OpenAI、Anthropic | 简单易用，质量持续提升 | 绑定单一厂商，结构支持有限 |
| Re-prompting 库 | Instructor、LangChain | 跨厂商兼容，支持正则约束 | 重试有成本，不保证成功 |
| 结构化生成（Constrained Decoding） | Outlines、SGLang、Guidance、XGrammar | 速度最快、质量最高、支持开源模型 | 需要控制 logits，无法用于闭源 API |

**结构化生成工作原理：**
1. 正常生成下一个 token 的 logit 分布
2. 将不符合结构约束的 token 概率置零
3. 重新归一化剩余概率后采样
4. 重复直到生成完整合法输出

## 关键结论

- 手写解析器（Parser）处理 LLM 输出不可扩展——一旦需求变化（如新增"是否投诉"字段）就会变得极其混乱
- JSON 结构化输出让代码从"解析文本"变成"读取字段"，是构建多步骤 Agent 的前提
- 结构化生成在多项评测中比非结构化方式表现更好，且推理成本几乎为零
- 支持的输出格式不止 JSON：正则表达式、语法正确的代码都可以作为约束目标
